# 零基础：问卷后台与数据分析怎么用

你不需要会写代码。日常只做三件事：**收问卷 → 导出表格 → 用 Excel/飞书做透视**。

当前问卷版本：**v23**。线上历史测试答卷已清空，从现在起收的都是正式投放样本。

---

## 1. 用户怎么交问卷

- 把问卷链接发给用户（微信群、海报、私聊均可）
- 用户打开手机浏览器填写并提交
- 每份答卷自动存到 Cloudflare D1 数据库的 `responses` 表

**线上地址（海外/代理可开）：**  
https://xolome-market-survey.xueyingwang1230.workers.dev/

**国内推荐（需运维在 api.xolome.com 配好反代后）：**  
https://api.xolome.com/survey/  
配置说明见 `deploy/nginx-survey-china.conf`  
（若尚未上服务器，大陆可能打不开 `*.workers.dev`，属网络环境问题，不是问卷坏了。）

## 2. 你怎么当「管理员」导出数据

后台页**没有复杂列表界面**，核心是「密钥 → 导出」。

1. 打开管理页：  
   https://xolome-market-survey.xueyingwang1230.workers.dev/admin  
   （国内反代后：`https://api.xolome.com/survey/admin`）
2. 输入管理密钥（Cloudflare Secret：`SURVEY_ADMIN_TOKEN`，勿写进代码仓库）
3. 点 **导出 CSV（推荐分析）** → 用 Excel / WPS / 飞书打开  
4. 需要跟进联系人时，再勾选「包含称呼 / 联系方式」后导出

也可直接浏览器打开（把 `TOKEN` 换成密钥）：

```
.../api/export?token=TOKEN&format=csv
.../api/export?token=TOKEN&format=csv&includeContact=1
.../api/export?token=TOKEN&format=json
.../api/stats?token=TOKEN
```

| 参数 | 作用 |
|------|------|
| `format=csv` | Excel 友好，默认**中文表头**，选项已译成中文 |
| `format=json` | 程序用，英文字段名（flat 键）不变 |
| `headers=en` | CSV 改用英文字段名（脚本兼容） |
| `includeContact=1` | 带上称呼/联系方式（默认不含，更安全） |

**筛版本：** CSV 里有「问卷版本」列（如 `v23`）。在 Excel 用筛选/透视按版本分开看；不要把改题前后的版本直接混加。

**看样本量：** 打开 `/api/stats?token=TOKEN`，返回当前答卷总数与线上 schema 版本。

## 3. 数据存在哪、长什么样

| 环境 | 存哪 |
|------|------|
| 线上（Cloudflare Worker） | D1 数据库 `xolome-survey` → 表 `responses` |
| 本地开发 | `data/survey.sqlite` → 表 `responses` |

每行主要字段：

- `id`：答卷 ID  
- `version`：提交时的问卷版本（如 v23）  
- `answers`：原始答题 JSON（题号 → 选项 value）  
- `flat`：分析用扁平字段（英文字段键，如 `digitalBudget`、`buyTiming`）  
- `created_at`：提交时间（UTC）

导出时：

- **CSV**：中文表头（答卷ID、月度数码兴趣预算…）+ 选项已译成中文标签  
- **JSON**：保留英文字段键，适合脚本；选项仍是英文 value

## 4. 零基础市场分析：建议路径

### 4.1 先做基础盘点

用 Excel「插入 → 数据透视表」或飞书「透视表」：

1. **样本量**：导出行数 ≈ 有效答卷数  
2. **完成结构**：年龄 × 性别 → 谁在填  
3. **单题分布**：各题计数（功能偏好、价位、渠道…）

### 4.2 重点交叉（对定价/渠道/话术更有用）

| 交叉 | 看什么 | 可能指导 |
|------|--------|----------|
| **预算(Q3) × 想用程度/摆放(Q12)** | 有预算的人是否真想摆机 | 定价档与目标客群是否匹配 |
| **场景(Q7) × 整机价位(Q13)** | 哪类场景更能接受更高价 | 场景化卖点与套餐 |
| **IP 兴趣门控后的偏好(Q9→Q10→11*)** | 只有对某功能感兴趣的人，后续偏好 | IP/外壳/数字内容是否值得投入 |
| **入手时机(Q15)** | 马上买 / 等评价 / 观望 | 预售节奏与内容投放 |

再补两张常用表：

5. **购买漏斗**：体验资格意向(Q14) × 入手时机(Q15)  
6. **观望卡点(Q14a) + 了解渠道(Q17)**：营销该补什么、钱往哪投

### 4.3 哪些结论能用、哪些不能

**能指导产品/运营（在样本够大时）：**

- 相对排序：哪类功能更常被选「最想先试」  
- 价位集中区间（剔除「根本不想用」后再看）  
- 渠道占比 → 投放倾斜  

**不能直接当「全体用户真相」：**

- **小样本**（例如 <50）：波动大，只作假设  
- **自选择偏差**：愿意点链接填写的人 ≠ 全体潜客；本问卷已无券，但仍偏「愿意配合调研」的人  
- **版本混加**：v21/v22/v23 选项可能变了，按「问卷版本」筛选  

### 4.4 和产品决策的简单对应

| 你看到的信号 | 可考虑的动作 |
|--------------|--------------|
| 高预算 × 高想用集中 | 优先打这群人的价与话术 |
| 某场景价位显著更高 | 场景化包装 / 套装 |
| IP 门控后仍强兴趣 | 值得做 IP 合作与可换外壳 |
| 「等评测 / 不确定」多 | 补真人评测、开箱、对比内容 |
| 「根本不想用」占比高 | 先查认知错位，再谈转化 |

## 5. 日常怎么管

| 频率 | 做什么 |
|------|--------|
| 每周 | 导出 CSV，备份到飞书/网盘 |
| 改题后 | 升版本号（v24…）；旧数据按旧版本分析 |
| 联系方式 | 默认不导出；需要跟进时再勾选 |
| 密钥 | 只给运营负责人；可在 Cloudflare 改 `SURVEY_ADMIN_TOKEN` |

清空测试数据：Cloudflare D1 控制台执行 `DELETE FROM responses;`（正式投放前建议清空）。

---

## 6. 记住一句话

**链接给人填 → `/admin` 导出 CSV → Excel 透视表看分布与交叉。**  
技术细节（Worker、D1、schema）先不管；要改题或部署时再找开发即可。
